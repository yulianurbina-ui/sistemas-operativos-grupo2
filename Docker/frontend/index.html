Conclusiones (Entrega 1)

💡 Aprendizajes Obtenidos

Durante esta primera fase, el equipo logró consolidar la administración de sistemas Linux mediante el particionamiento manual avanzado (/, swap, /home), garantizando el aislamiento de datos y optimización de recursos. Asimismo, se comprendió el ciclo de vida de la contenerización al construir imágenes eficientes desde cero con Docker Engine, y la importancia de la automatización multi-contenedor usando Docker Compose para el despliegue ordenado de arquitecturas de software.

⚠️ Dificultades y Retos Superados

Una de las mayores dificultades que se afronto en esta primera entrega, es el manejo de linux, que para muchos de los integrantes del equipo, a la hora de la instalacion y realizacion de su respectivo componente, no se tenia tanto conocimiento para la movilidad en dicho Sistema Operativo, otra de ellas fueron la asignación de IPs estáticas y permisos en el servicio SSH para asegurar una comunicación aislada pero funcional.
